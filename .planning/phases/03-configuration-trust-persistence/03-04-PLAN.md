```rust
// -- Trust cache load (SEC-05) --
// Loads the fingerprint trust cache from the XDG data dir. If the file
// does not exist (first run), starts with an empty cache. The cache is
// only written to via AcceptFingerprint IPC (D-04).
let trust_path = periphore_trust::default_trust_path()
    .ok_or_else(|| anyhow::anyhow!("cannot determine trust cache storage path"))?;

    .map_err(|e| anyhow::anyhow!("trust cache error: {e}"))?;
tracing::info!(path = %trust_path.display(), "trust cache loaded");
```

Note: `trust_store` must be `mut` because `add_trusted` requires `&mut self`.

**2. Add named match arms for AcceptFingerprint and RejectFingerprint** in the main `tokio::select!` loop. Insert them after the existing `GetWordPhrase` arm (after line 163) and before the `ReloadConfig` arm:

```rust
Some(IpcCommand::AcceptFingerprint { fingerprint, responder }) => {
    tracing::info!(fingerprint = %fingerprint, "IPC: AcceptFingerprint");
    match trust_store.add_trusted(&fingerprint, None, &trust_path) {
        Ok(()) => {
            tracing::info!(fingerprint = %fingerprint, "fingerprint trusted and cached");
            let _ = responder.send(IpcResponse::Ok);
        }
        Err(e) => {
            tracing::error!(error = %e, "failed to cache trusted fingerprint");
            let _ = responder.send(IpcResponse::Error {
                message: format!("trust cache error: {e}"),
            });
        }
    }
}
Some(IpcCommand::RejectFingerprint { fingerprint, responder }) => {
    // Rejection is stateless â€” no state change needed.
    // The daemon simply does not add the fingerprint to the trust cache.
    tracing::info!(fingerprint = %fingerprint, "IPC: RejectFingerprint (no state change)");
    let _ = responder.send(IpcResponse::Ok);
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IPC client -> daemon | AcceptFingerprint/RejectFingerprint commands cross the Unix socket boundary; client is local (same machine, same user) |
| daemon -> filesystem | TrustStore writes trusted.toml to disk via atomic tempfile::persist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-09 | Spoofing | AcceptFingerprint IPC | accept | IPC socket has 0o600 permissions (Phase 1); only local processes running as the same user can send commands; no remote attack vector |
| T-3-10 | Tampering | TrustStore mutation via IPC | mitigate | Only `AcceptFingerprint` mutates trust state; `RejectFingerprint` is explicitly stateless; sequential processing via tokio::select! prevents concurrent mutation (RESEARCH.md Pitfall 5) |
| T-3-11 | Repudiation | Trust decision logging | mitigate | Both `AcceptFingerprint` and `RejectFingerprint` log at `info` level with fingerprint value; trust additions are logged with "fingerprint trusted and cached"; errors logged at `error` level |
| T-3-12 | Denial of Service | Trust cache startup load failure | mitigate | `TrustError` is mapped to `anyhow::Error` at daemon boundary; daemon exits with clear error message rather than starting with corrupt state; `CorruptCacheFile` prevents silent acceptance of bad data |

</threat_model>
