**2e. net_event arm in select!** — add as a new branch in the select! loop, alongside the existing signal and IPC arms:

// TCP peer event from ConnectionManager
net_event = net_event_rx.recv() => {
    match net_event {
        Some(periphore_net::PeerEvent::PeerPending { fingerprint, identicon, word_phrase }) => {
            // D-02: log at WARN so operator sees the pending peer in daemon output
            tracing::warn!(
                fingerprint = %fingerprint,
                "unknown peer pending verification — run: periphore trust accept {}",
                &fingerprint[..fingerprint.len().min(16)]
            );
            if !identicon.is_empty() {
                tracing::warn!("peer identicon:\n{identicon}");
            }
            if !word_phrase.is_empty() {
                tracing::warn!(phrase = %word_phrase.join(" "), "peer word phrase");
            }
        }
        Some(periphore_net::PeerEvent::PeerConnected { peer_id }) => {
            tracing::info!(peer_id = %peer_id, "peer connected and trusted");
        }
        Some(periphore_net::PeerEvent::PeerDisconnected { peer_id }) => {
            tracing::info!(peer_id = %peer_id, "peer disconnected");
            // Return focus if we were forwarding to this peer
            let _ = focus_sm.reclaim();
        }
        None => {
            tracing::warn!("net event channel closed");
        }
    }
}

**2f. AcceptFingerprint — extend with promote_pending** — replace the existing AcceptFingerprint arm:

```rust
Some(IpcCommand::AcceptFingerprint { fingerprint, responder }) => {
    tracing::info!(fingerprint = %fingerprint, "IPC: AcceptFingerprint");
    match trust_store.write().unwrap().add_trusted(&fingerprint, None, &trust_path) {
        Ok(()) => {
            tracing::info!(fingerprint = %fingerprint, "fingerprint trusted and cached");

            tracing::error!(error = %e, "failed to cache trusted fingerprint");
            let _ = responder.send(IpcResponse::Error {
                message: format!("trust cache error: {e}"),
            });
        }
    }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| macOS stdin → daemon startup | Daemon must not start headlessly on macOS over SSH (CGEvent requires local session) |
| IPC client → AcceptFingerprint | AcceptFingerprint now promotes network connections — trust store write + network state promotion coupled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-06 | Denial of Service | periphored/src/main.rs macOS SSH check | mitigate | `#[cfg(target_os = "macos")] { if !stdin().is_terminal() { eprintln!(...); exit(1); } }` at top of main() before any async setup. Linux unaffected (cfg-gated). |
| T-6-02 | Elevation of Privilege | GetPendingVerifications IPC + AcceptFingerprint | mitigate | GetPendingVerifications returns data only; AcceptFingerprint is the only promotion path, requiring explicit user action. No auto-promotion. |
| T-6-04 | Spoofing | AcceptFingerprint + promote_pending coupling | accept | AcceptFingerprint first writes to trust store (persistent), then calls promote_pending (best-effort). If promote_pending fails (peer already reconnected), the next handshake finds the fingerprint in trust store and auto-connects. Not a security gap — trust store is the authority. |
</threat_model>
