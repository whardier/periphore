**Change 1: crates/periphore-config/src/schema.rs**

Add the `listen` field to `DaemonConfig` struct. Insert after the `port` field:

```rust
/// Whether the daemon listens for incoming peer TCP connections.
/// Default: true (P2P symmetric model, D-07).
/// Set to false for CI/testing setups that should not accept incoming peers.
#[serde(default = "default_listen")]

```

Add the default function after the `DaemonConfig` struct closing brace (before `LoggingConfig`):

```rust
fn default_listen() -> bool {
    true
}
```

The `DaemonConfig` derives `Default` via `#[derive(Default)]`. With `listen` being a `bool` field with `#[serde(default = "default_listen")]`, the `Default` derive will set it to `false` (bool default). This is WRONG â€” Default must return `true`. Fix: remove `Default` from the derive and add a manual `Default` impl, OR add `default_listen` to the derive by using a serde attribute only for serde (not for Default). The correct approach: replace `#[derive(Debug, Deserialize, Default)]` with `#[derive(Debug, Deserialize)]` and add a manual `impl Default for DaemonConfig`. Use the existing pattern from `LoggingConfig::default()` and `IdentityConfig::default()` in the same file:

```rust
#[derive(Debug, Deserialize)]
pub struct DaemonConfig {
    pub socket_path: Option<std::path::PathBuf>,
    pub port: Option<u16>,
    #[serde(default = "default_listen")]
    pub listen: bool,
}

impl Default for DaemonConfig {
    fn default() -> Self {
        Self {
            socket_path: None,
            port: None,
            listen: true,
        }
    }
}

fn default_listen() -> bool {
    true
}
```
