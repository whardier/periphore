1. Receive `PeerMessage::Hello { protocol_version, fingerprint, public_key }` with timeout
2. Convert fingerprint to hex
3. Build `peer_id = PeerId::new(peer_fp_hex.clone())`

7. Check trust store → return `HandshakeResult::Trusted` or `HandshakeResult::Pending` (same pattern as initiator)

`fn spawn_listener(&mut self, tasks: &mut JoinSet<anyhow::Result<()>>, bind_addr: SocketAddr, identity: Arc<IdentityStore>, trust_store: Arc<RwLock<TrustStore>>)` — binds a TcpListener and spawns an accept loop task. The accept loop:
  - Calls `listener.accept().await`
  - On Ok((stream, peer_addr)): IMMEDIATELY call `stream.set_nodelay(true)` — if Err, log ERROR with peer_addr and continue to next accept
  - After set_nodelay succeeds: call `codec::split_framed(stream)` to get (framed_read, framed_write)
  - Spawn a task: call `handshake::perform_handshake_responder(&mut framed_read, &mut framed_write, &identity, &trust_store, None).await`
  - Based on HandshakeResult, send appropriate PeerEvent via event_tx (clone per spawn)
  - On err accept: log ERROR and continue (mirror server.rs pattern — never abort the loop on a single error)

`fn spawn_connector(&mut self, tasks: &mut JoinSet<anyhow::Result<()>>, peer_config: PeerConfig, identity: Arc<IdentityStore>, trust_store: Arc<RwLock<TrustStore>>)` — creates a CancellationToken, stores it in peer_tokens keyed by `peer_config.name.clone().unwrap_or_else(|| peer_config.host.clone().unwrap_or_default())`, spawns the retry loop task.

The retry loop (inside the spawned task):
```
let mut delay_ms = BACKOFF_INITIAL_MS;
loop {
    // try connect
    match TcpStream::connect(addr).await {
        Ok(stream) => {
            // HARD REQUIREMENT: TCP_NODELAY immediately after connect
            if let Err(e) = stream.set_nodelay(true) {
                tracing::error!(addr = %addr, error = %e, "TCP_NODELAY failed — dropping");
                // fall through to retry
            } else {
                delay_ms = BACKOFF_INITIAL_MS; // reset on success
                let (mut framed_read, mut framed_write) = codec::split_framed(stream);
                match handshake::perform_handshake_initiator(&mut framed_read, &mut framed_write, &identity, &trust_store, Some(&peer_config)).await {
                    Ok(HandshakeResult::Trusted { peer_id, .. }) => {
                        event_tx.send(PeerEvent::PeerConnected { peer_id: peer_id.clone() }).await.ok();
                        // connection active — wait for it to drop (Phase 9 adds the active read loop)
                        // For Phase 6: just hold the connection open until EOF/error
                        // Read loop: wait for next frame; on ConnectionClosed or error, break
                        loop {
                            match tokio::time::timeout(Duration::from_secs(30), framed_read.next()).await {
                                Ok(Some(Ok(_frame))) => { /* ignore non-handshake frames in Phase 6 */ }
                                _ => break,
                            }
                        }
                        event_tx.send(PeerEvent::PeerDisconnected { peer_id }).await.ok();
                    }
                    Ok(HandshakeResult::Pending { peer_id, fingerprint_hex, identicon, word_phrase }) => {
                        // Notify daemon; hold in pending; wait for ConnectionControl
                        // promote_tx/rx pair created per pending connection
                        let (promote_tx, mut promote_rx) = mpsc::channel::<ConnectionControl>(1);
                        // Send PeerPending event to daemon
                        event_tx.send(PeerEvent::PeerPending { fingerprint: fingerprint_hex.clone(), identicon: identicon.clone(), word_phrase: word_phrase.clone() }).await.ok();
                        // Store pending entry (needs event_tx to be accessible — use a local channel approach)
                        // NOTE: ConnectionManager::pending HashMap is in the manager struct,
                        // not accessible from inside the spawned task.
                        // For Phase 6: send promote_tx with the PeerPending event via a second channel,
                        // OR store the promote_tx in a shared Arc<Mutex<HashMap>>.
                        // Simplest correct approach: use an Arc<Mutex<HashMap<String, PendingPeer>>>
                        // shared between ConnectionManager and spawned tasks.
                        // Wait for promote/reject signal
                        match promote_rx.recv().await {
                            Some(ConnectionControl::PromoteTrusted) => {
                                event_tx.send(PeerEvent::PeerConnected { peer_id }).await.ok();
                            }
                            _ => {} // rejected or channel dropped
                        }
                    }
                    Ok(HandshakeResult::Rejected { reason }) => {
                        tracing::warn!(reason = %reason, "handshake rejected — will retry");
                    }
                    Err(e) => {
                        tracing::info!(addr = %addr, error = %e, "handshake failed — will retry");
                    }
                }
            }
        }
        Err(e) => {
            tracing::info!(addr = %addr, error = %e, delay_ms, "peer connection failed — retrying");
        }
    }
    // Check cancellation before sleep
    tokio::select! {
        _ = token.cancelled() => { return Ok(()); }
        _ = tokio::time::sleep(Duration::from_millis(delay_ms)) => {}
    }
    delay_ms = (delay_ms * 2).min(BACKOFF_CAP_MS);
}
```

pub async fn promote_pending(&self, fingerprint_hex: &str) -> Result<(), NetError> {
    let tx = {
        let guard = self.pending.lock().map_err(|_| NetError::PeerNotFound(fingerprint_hex.to_owned()))?;
        guard.get(fingerprint_hex).map(|p| p.promote_tx.clone())
    };
    match tx {
        Some(ch) => {
            ch.send(ConnectionControl::PromoteTrusted).await
                .map_err(|_| NetError::PeerNotFound(fingerprint_hex.to_owned()))
        }
        None => {
            // Peer not in pending — may have already connected or reconnected.
            // Log at INFO (not error) per RESEARCH.md Pitfall 4.
            tracing::info!(fingerprint = %fingerprint_hex, "promote_pending: peer not in pending state (may have already reconnected as trusted)");
            Ok(())
        }
    }
}
