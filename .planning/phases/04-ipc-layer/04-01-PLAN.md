<objective>
CR-01 fix: eliminate the 100% CPU spin when the IPC task exits cleanly.

The bug: `JoinSet::join_next()` returns `Poll::Ready(None)` immediately when the set is
empty. Inside `tokio::select!`, this makes the `join_next` branch always win when there
are no tasks, spinning the loop at 100% CPU.

Two targeted edits to `crates/periphored/src/main.rs`:
1. Add `, if !tasks.is_empty()` precondition to the `join_next` branch — when the set
   is empty this branch is disabled and other branches (signals, IPC channel) remain live.
2. Add `break;` on `Some(Ok(Ok(())))` — when the IPC server exits cleanly the daemon
   shuts down instead of spinning on an empty set.

No new files. No new deps. This is a two-line surgical fix.
</objective>
