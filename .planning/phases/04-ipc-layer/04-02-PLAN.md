<objective>
Config reload: implement SIGHUP + ReloadConfig IPC so the daemon picks up new config
values from disk without restarting.

Scope:
1. Restructure the tracing subscriber init to use `tracing_subscriber::reload::Layer`
   so the log filter can be updated at runtime (D-03).
2. Extract a `do_reload_config` helper that loads config from disk, swaps the in-memory
   binding, reloads the tracing filter, and logs warnings for restart-required fields.
3. Wire the helper into the SIGHUP arm (previously a placeholder) and the ReloadConfig
   IPC arm.

Restart-required fields (D-02): `daemon.socket_path`, `daemon.port`. Changes to these
fields are logged as `warn` but NOT applied. All other fields take effect immediately.

Reload failure (D-04): on `periphore_config::load()` error, log the error at `error`
level and keep the current config. Respond `IpcResponse::Error` to the IPC client.

Identity and trust store are NOT reloaded on SIGHUP/ReloadConfig (D-05).
</objective>

In `crates/periphored/src/main.rs`, implement the config reload logic.

**Part A — make `config` a mutable binding:**
The `config` binding loaded at startup must be `mut` so it can be replaced on reload:
```rust
    let mut config = periphore_config::load(args.config.as_deref())
        .map_err(|e| anyhow::anyhow!("failed to load config: {e}"))?;
```

**Part B — define a reload helper closure or inline block:**
Rather than a standalone function (which complicates borrow rules with `config` and
`filter_handle`), implement the reload logic inline at each call site, or extract a
free function that takes all the pieces as arguments. The cleanest pattern for Rust:
a free function `reload_config` that takes `config_path`, `current_config`, and
`filter_handle` by reference:

```rust
/// Reload config from disk and update the daemon's live state.
///
/// Returns the new Config on success. Returns None on failure (error already logged).
/// The caller must replace its `config` binding with the returned value.
fn reload_config(
    config_path: Option<&std::path::Path>,
    current_config: &periphore_config::Config,
    filter_handle: &tracing_subscriber::reload::Handle<
        tracing_subscriber::EnvFilter,
        impl tracing::Subscriber,
    >,
) -> Option<periphore_config::Config> {
    match periphore_config::load(config_path) {
        Ok(new_config) => {
            // Hot-reload logging level (D-03)
            if new_config.logging.level != current_config.logging.level {
                match tracing_subscriber::EnvFilter::try_new(&new_config.logging.level) {
                    Ok(new_filter) => {
                        if let Err(e) = filter_handle.reload(new_filter) {
                            tracing::warn!(error = %e, "failed to reload tracing filter");
                        } else {
                            tracing::info!(
                                level = %new_config.logging.level,
                                "logging level reloaded"
                            );
                        }
                    }
                    Err(e) => {
                        tracing::warn!(error = %e, "invalid logging level in new config — keeping old filter");
                    }
                }
            }

            // Restart-required fields: log warn if changed (D-02)
            if new_config.daemon.socket_path != current_config.daemon.socket_path {
                tracing::warn!(
                    "config field 'daemon.socket_path' changed but requires restart to take effect"
                );
            }
            if new_config.daemon.port != current_config.daemon.port {
                tracing::warn!(
                    "config field 'daemon.port' changed but requires restart to take effect"
                );
            }

            tracing::info!("config reloaded successfully");
            Some(new_config)
        }
        Err(e) => {
            tracing::error!(error = %e, "config reload failed — keeping existing config");
            None
        }
    }
}
```

Note on the `impl tracing::Subscriber` generic in the Handle type: the exact type parameter
for `Handle` depends on how `registry()` was constructed. If the compiler rejects the generic
bound, use `tracing_subscriber::reload::Handle<tracing_subscriber::EnvFilter, _>` with a
type alias, or move the reload logic inline in the two call sites instead. Pick whichever
compiles cleanly.

With:
```rust
                    Some(IpcCommand::ReloadConfig { responder }) => {
                        tracing::info!("IPC: ReloadConfig");
                        match reload_config(args.config.as_deref(), &config, &filter_handle) {
                            Some(new_config) => {
                                config = new_config;
                                let _ = responder.send(IpcResponse::Ok);
                            }
                            None => {
                                let _ = responder.send(IpcResponse::Error {
                                    message: "config reload failed — see daemon logs".to_owned(),
                                });
                            }
                        }
                    }
```

**Important:** The borrow checker may complain because `config` is borrowed immutably
by `reload_config` and then mutated in the match arm. Fix: pass `&config` before the
assignment. This is valid in Rust — `reload_config` returns an owned `Option<Config>`,
the borrow ends when the call returns, then `config = new_config` is a fresh assignment.

If the `reload_config` free function has trouble with the `Handle` type parameter, move
the reload logic inline into each arm. Correctness matters more than code deduplication.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config file | Reloaded from disk on SIGHUP or IPC command — file content is user-controlled |
| IPC socket | ReloadConfig caller must have access to the Unix domain socket (already 0600) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-4-02 | Tampering | Config file on reload | accept | Config file is user-owned; same trust level as startup load; no privilege escalation |
| T-4-03 | DoS | Repeated ReloadConfig IPC spam | accept | Reload is synchronous in the event loop; each reload holds the loop for ~1ms; acceptable for a local IPC tool |
| T-4-04 | Info Disclosure | Error messages in IpcResponse::Error | accept | Errors are from periphore_config::load() on a local file — no secrets exposed in error strings |

</threat_model>
